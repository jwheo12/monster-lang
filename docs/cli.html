CLI Reference

The mst command is the entrypoint for checking, building, running, and cleaning Monster projects.

Basic Commands

mst check exam.mnst
mst emit-llvm exam.mnst
mst build exam.mnst
mst run exam.mnst
mst clean
mst -upgrade
mst --help
mst --version

Debug Builds

mst build --debug exam.mnst
mst run --debug exam.mnst

Debug builds keep the raw LLVM IR in target/mst/, skip LLVM -O2, and link with clang -g -O0.

Passing Program Arguments

mst run examples/argv.mnst -- hello from monster

Everything after -- is forwarded to the compiled Monster program. Monster programs can receive these through main(argc: i32, argv: **u8).

Build Output

Generated binaries and intermediate LLVM files live in target/mst/. This keeps project roots clean while preserving artifacts useful for debugging.

Update Compiler

mst -upgrade

This downloads the latest published GitHub Release and installs it back into the current mst install directory. Global installs may ask for sudo.

Pipeline
