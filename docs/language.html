Language Overview

Monster currently blends a Rust-leaning surface syntax with low-level C-style control, pointers, explicit memory work, and an LLVM-first backend.

Functions and Control Flow

fn classify(x: i32) -> i32 {
    if x < 0 {
        return -1;
    } else if x == 0 {
        return 0;
    } else {
        return 1;
    }
}
  • fn and extern fn
  • return expr; and return;
  • if, else, while, break, continue

Types

Built-in scalar types:

  • i32
  • u8
  • usize
  • bool
  • str
  • void

Composite types and low-level forms:

  • fixed arrays: [T; N]
  • slices: [T] and slice(array)
  • raw pointers: *T, &expr, *ptr, ptr[i]
  • structs
  • enums and payload enums

Structs

struct Pair {
    left: i32,
    right: i32,
}

fn main() -> i32 {
    let mut pair: Pair = Pair { left: 10, right: 20 };
    pair.left = 42;
    return pair.left + pair.right;
}

Enums and Match

enum Token {
    Int(i32),
    Name(str),
    Eof,
}

fn value(token: Token) -> i32 {
    return match token {
        Int(n) => n,
        Name(_) => 0,
        Eof => -1,
    };
}

Current match support is enum-only and expression-based. Arms must be exhaustive, and payload variants bind through Variant(name) patterns.

Imports and Modules

import "math.mnst" as math;

fn main() -> i32 {
    return math.add(3, 4);
}

Imports are file-based today. Alias-qualified calls like math.add(...) help keep larger programs organized.

Builtins

  • printing: print_*, print_ln_*
  • input: read_i32
  • array and slice helpers: len, slice
  • file I/O: read_file, write_file
  • string and byte helpers: strlen, memcmp, memcpy, str_eq
  • type utility: sizeof(T)

Current flavor

Monster is still intentionally small. The syntax leans Rust-like, while the execution model stays close to C and LLVM.
