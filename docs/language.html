Language Reference

This page is a feature-by-feature guide to the current Monster language. It focuses on what exists today and how each piece is meant to be used.

Functions

Functions are declared with fn. External functions use extern fn.

fn classify(x: i32) -> i32 {
    if x < 0 {
        return -1;
    } else if x == 0 {
        return 0;
    } else {
        return 1;
    }
}
  • Parameters use name: type.
  • Return types use -> type.
  • return expr; and return; are both supported.
  • Non-void functions must return on all reachable paths.

Control Flow

Monster currently supports conditional logic and loops in a direct, expression-light style.

  • if / else
  • while
  • break and continue inside loops
  • match as an enum-focused expression

Constants, Variables, and Assignment

Global constants use const. Local bindings use let, and reassignment requires mut.

const LIMIT: usize = 64 as usize;

let x: i32 = 10;
let mut y: i32 = 20;
y = y + x;
  • Constants live at the top level and are inlined into function bodies.
  • Constant initializers currently support compile-time scalar expressions such as literals, arithmetic, casts, booleans, strings, and sizeof(T).
  • Local variable declarations are typed explicitly.
  • Immutable variables reject assignment in semantic analysis.
  • Array element, struct field, and dereference assignment are supported.

Types

Type What it is for
i32 signed 32-bit integer
u8 byte-sized unsigned integer
usize pointer-sized unsigned integer, used for lengths and sizes
bool boolean value
str string-like runtime pointer used with builtins and C-style string flows
void no return value

Monster also supports type casts with as and size queries with sizeof(T).

Arrays and Slices

let values: [i32; 3] = [10, 20, 30];
let view: [i32] = slice(values);
let n: usize = len(view);
  • Fixed arrays use [T; N].
  • Slices use [T] and can be built from arrays with slice(array).
  • Reading and writing through indexing are both supported.
  • Nested array index assignment is supported.

Pointers

let p: *i32 = &values[0];
*p = 42;
let next: i32 = p[1];
  • Pointer types use *T.
  • Address-of uses &expr.
  • Dereference reads use *ptr.
  • Dereference writes and pointer indexing are supported.

Structs

struct Pair {
    left: i32,
    right: i32,
}

fn main() -> i32 {
    let mut pair: Pair = Pair { left: 10, right: 20 };
    pair.left = 42;
    return pair.left + pair.right;
}
  • Struct declarations live at the top level.
  • Struct literals use named fields.
  • Field reads and field assignment are supported.

Enums and Match

enum Token {
    Int(i32),
    Name(str),
    Eof,
}

fn value(token: Token) -> i32 {
    return match token {
        Int(n) => n,
        Name(_) => 0,
        Eof => -1,
    };
}

Current match support is enum-only and expression-based.

  • Payload-free enums are supported.
  • Single-payload variants like Int(i32) are supported.
  • match arms must be exhaustive.
  • Payload variants bind through Variant(name) patterns.

Imports and Modules

import "math.mnst" as math;

fn main() -> i32 {
    return math.add(3, 4);
}

Imports are file-based today. Alias-qualified calls like math.add(...) help keep larger programs organized.

  • import "path.mnst";
  • import "path.mnst" as name;
  • Qualified calls currently apply to functions.

Program Entrypoint and Arguments

fn main(argc: i32, argv: **u8) -> i32 {
    if argc > 1 {
        print_ln_str(argv[1] as str);
    }

    return 0;
}

Monster programs can accept command-line arguments directly through main(argc, argv).
