Install Monster

Use the release installer for the fastest setup, or install from source when you want to work on the compiler itself.

Release Installer

Linux and macOS:

curl -fsSL https://raw.githubusercontent.com/BitIntx/monster-lang/main/install/install-release.sh | bash
mst --help

Windows PowerShell:

irm https://raw.githubusercontent.com/BitIntx/monster-lang/main/install/install-release.ps1 | iex
mst --version

System-wide install on Unix-like systems:

curl -fsSL https://raw.githubusercontent.com/BitIntx/monster-lang/main/install/install-release.sh | sudo env PREFIX=/usr/local bash
mst --version

Pin a Specific Release

curl -fsSL https://raw.githubusercontent.com/BitIntx/monster-lang/main/install/install-release.sh | env MST_VERSION=v0.1.0-rc3 bash
$env:MST_VERSION = "v0.1.0-rc3"
irm https://raw.githubusercontent.com/BitIntx/monster-lang/main/install/install-release.ps1 | iex

Build From Source

git clone https://github.com/BitIntx/monster-lang.git
cd monster-lang
./install/install.sh
mst --help

The compiler itself is written in Rust. Building from source is the right choice if you want to hack on Monster, test changes locally, or package custom builds.

Toolchain Requirements

For mst build and mst run, you still need clang-18 or clang, and opt-18 or opt on your PATH.

The release installer currently supports Linux x86_64, Linux ARM64, macOS x86_64, macOS arm64, and Windows x86_64.

Update

The release installer also doubles as the update path:

curl -fsSL https://raw.githubusercontent.com/BitIntx/monster-lang/main/install/install-release.sh | bash

Default install path

Unix-like systems install into ~/.local/bin by default. Windows installs into %LOCALAPPDATA%\Programs\mst\bin.
