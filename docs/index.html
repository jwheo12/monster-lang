Monster documentation

Reference and guides for the Monster compiler and language.

Monster is an experimental low-level, ahead-of-time systems language. These docs are meant to stay close to the implementation: straightforward pages, feature lists, and runnable examples.

Pipeline

.mnst source
  -> lexer
  -> parser
  -> AST
  -> semantic analysis
  -> LLVM IR
  -> opt-18 (-O2)
  -> clang-18
  -> native binary
  • Compiler executable: mst
  • Source extension: .mnst
  • Docs site: suitable for GitHub Pages and editor help links

Current Feature Areas

  • Core syntax: fn, extern fn, const, let, let mut, if, while, break, continue, return
  • Types: i32, u8, usize, bool, str, void
  • Data structures: arrays, slices, structs, enums, payload enums, pointers
  • Utilities: sizeof(T), casts with as, argc/argv, file I/O, string and byte helpers
  • Tooling: LLVM IR emission, optimized builds, debug builds, static docs, a VS Code extension, and early self-hosting experiments

Self-Hosting

The first self-hosting slice lives in selfhost/. It is a Monster-written lexer prototype that reads Monster source with read_file and builds a token buffer with token kinds close to the Rust lexer. 

mst run selfhost/main.mnst -- exam.mnst
